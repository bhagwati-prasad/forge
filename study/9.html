Encyclopedia of Two-Pointer Techniques

1. Converging Pointers (Opposite Ends)
Movement: Pointers start at indices 0 and n-1 and move toward each other.
Key Insight: Commonly used on sorted data to prune the search space from both ends simultaneously based on a comparison.
Common Applications:
  • Two Sum (Sorted Array), Valid Palindrome, Container With Most Water, Reversing an Array
2. Slow-Fast Pointers (Tortoise and Hare)
Movement: Both pointers move in the same direction, but one moves at double the speed of the other.
Key Insight: Ideal for detecting cycles in a structure or finding structural midpoints in a single pass.
Common Applications:
  • Linked List Cycle Detection, Finding the Middle of a Linked List, Happy Number
3. Sliding Window (Fixed or Variable)
Movement: Both pointers move in the same direction; the leading pointer expands the window while the trailing pointer shrinks it.
Key Insight: Converts nested loops into a single pass to find a subarray or substring meeting specific criteria.
Common Applications:
  • Longest Substring Without Repeating Characters, Minimum Size Subarray Sum, Maximum Sum Subarray of Size K
4. Diverging Pointers (Expand from Center)
Movement: Pointers start at the same center index and move outward toward the boundaries.
Key Insight: The most efficient way to check for symmetry or palindromic properties starting from the inside out.
Common Applications:
  • Longest Palindromic Substring, Palindromic Substrings Count
5. Multi-Structure Pointers
Movement: One pointer per array; they move independently based on logic comparisons.
Key Insight: Used to synchronize, merge, or compare two separate datasets efficiently.
Common Applications:
  • Merge Sorted Arrays, Intersection of Two Sorted Lists, K-th Smallest in Two Sorted Arrays
6. Read-Write Pointers (In-place)
Movement: Read scans all elements; Write only moves to record valid/modified data.
Key Insight: Enables in-place data transformation with O(1) space by overwriting existing memory.
Common Applications:
  • Remove Duplicates, Move Zeroes, String Compression
7. Saddleback Search (2D Pointers)
Movement: Starts at Top-Right; moves Left to decrease value or Down to increase value.
Key Insight: Eliminates an entire row or column per step in a 2D sorted matrix.
Common Applications:
  • Search in a 2D Sorted Matrix II
8. Binary Search (Logarithmic Pointers)
Movement: Pointers jump to the midpoint (mid) instead of stepping by 1.
Key Insight: Redefines boundaries to reduce the search space by half in each iteration.
Common Applications:
  • Search in Rotated Sorted Array, Search Insert Position
9. Interval Pointers (Sweep Line)
Movement: Two pointers moving across sorted start-times and end-times respectively.
Key Insight: Tracks overlaps and concurrent events by treating intervals as discrete events.
Common Applications:
  • Meeting Rooms II, Interval List Intersections
10. Triple Pointers (Partitioning)
Movement: Three pointers (Low, Mid, High) to manage three categories of data.
Key Insight: Enables linear time sorting of three distinct values in one pass.
Common Applications:
  • Dutch National Flag Problem (Sort Colors)
11. Cyclic Pointers (Index as Pointer)
Movement: The value at index i (nums[i]) becomes the next pointer location.
Key Insight: Treats array values as jump addresses to detect cycles or misplaced elements.
Common Applications:
  • Find the Duplicate Number, First Missing Positive
12. Two-Pass Greedy Pointers
Movement: A Left-to-Right pointer pass followed by a separate Right-to-Left pass.
Key Insight: Builds cumulative constraints from both directions to solve bilateral dependency.
Common Applications:
  • Trapping Rain Water, Candy Problem, Product of Array Except Self
