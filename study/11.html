Encyclopedia of Two-Pointer Techniques

1. Converging Pointers (Opposite Ends)
Two pointers start at the beginning and end of a structure and move inward.
Use Case: Finding pairs in sorted arrays or verifying symmetry.
Movement: Indices 0 and n-1 moving toward each other (left++, right--).
Key Insight: Eliminates search space in O(1) based on sorted properties.
Common Applications:
  • Two Sum Sorted
  • Valid Palindrome
  • Container With Water
  • 3Sum
  • Trapping Rain Water
  • Reverse Array
2. Slow-Fast Pointers (Tortoise and Hare)
Two pointers move forward at different speeds (1x and 2x).
Use Case: Cycle detection and finding midpoints without length metadata.
Movement: Both move forward; Fast skips one element (fast += 2).
Key Insight: Meeting point guarantees a cycle or exact midpoint ratio.
Common Applications:
  • Linked List Cycle
  • Middle of List
  • Happy Number
  • Cycle Start Node
  • Palindrome List
3. Sliding Window
Maintains a range between two pointers to track contiguous sub-segments.
Use Case: Tracking sum, uniqueness, or patterns in subarrays/substrings.
Movement: Right expands, Left contracts based on validity.
Key Insight: Incremental updates avoid re-calculating the whole window state.
Common Applications:
  • Longest Unique Substring
  • Max Sum Subarray
  • Permutation in String
  • Sliding Window Max
  • Min Window Substring
4. Diverging Pointers (Expand from Center)
Pointers start at a central pivot and expand outward.
Use Case: Identifying symmetry from the center-most point.
Movement: Opposite directions outward (left--, right++).
Key Insight: Stopping expansion immediately upon mismatch optimizes symmetry checks.
Common Applications:
  • Longest Palindrome
  • Shortest Palindrome
  • Count Sub-palindromes
  • Tree Symmetry
5. Multi-Structure Pointers
Independent pointers assigned to different data sources.
Use Case: Linear-time operations on multiple sorted streams.
Movement: Each pointer advances based on cross-structure comparisons.
Key Insight: Processes pre-sorted data without merging or re-sorting overhead.
Common Applications:
  • Merge Sorted Arrays
  • Intersection of Lists
  • K-th Smallest in Two Arrays
  • Is Subsequence
  • Merge Sort (Merge Step)
6. Read-Write Pointers (In-place)
One pointer iterates through the data; another tracks the modification index.
Use Case: Space-efficient in-place filtering or compression.
Movement: Both move forward; Write only advances on 'keep' conditions.
Key Insight: Safe in-place editing by ensuring Read index is always >= Write index.
Common Applications:
  • Remove Duplicates
  • Move Zeroes
  • String Compression
  • Remove Element
7. Saddleback Search (2D)
Navigation from a corner through a 2D sorted matrix.
Use Case: Rapidly finding a target in sorted 2D tables.
Movement: Left or Down depending on target comparison.
Key Insight: Discarding entire rows/columns reduces O(N*M) to O(N+M).
Common Applications:
  • Matrix Search II
  • Count Negative in Matrix
  • Young Tableau
8. Binary Search (Logarithmic)
Pointers define a range that is halved by jumping to the mid.
Use Case: Finding values or optimal solutions in sorted spaces.
Movement: Jump to midpoint (left + right) / 2.
Key Insight: Logarithmic efficiency achieved by dividing the search area.
Common Applications:
  • Sorted Search
  • Rotated Array Search
  • Peak Finding
  • BS on Answer
9. Interval Pointers
Pointers tracking separate sorted lists of start and end times.
Use Case: Overlap management and scheduling tasks.
Movement: Independent advance through Start and End event streams.
Key Insight: Overlap counting via discrete entrance/exit events.
Common Applications:
  • Meeting Rooms II
  • Interval Intersection
  • Merge Intervals
10. Triple Pointers (Partitioning)
Three pointers dividing an array into three distinct logical segments.
Use Case: Sorting arrays with a small set of unique values.
Movement: One scanning pointer with two boundary trackers.
Key Insight: Constant space partitioning of complex categories.
Common Applications:
  • Sort Colors
  • QuickSort Partition
  • 3-way Sort
11. Cyclic Pointers
Using the value found at an index as the next index to visit.
Use Case: Finding missing or duplicate items in range-bound arrays.
Movement: curr = nums[curr].
Key Insight: Maps array data as directed graphs to find cycle entry points.
Common Applications:
  • Duplicate Number
  • Missing Positive
  • Array Nesting
12. Two-Pass Greedy Pointers
Applying uni-directional pointer passes from both ends sequentially.
Use Case: Problems where local state depends on global bilateral context.
Movement: Left-to-Right pass then Right-to-Left pass.
Key Insight: Decouples complex multi-directional dependencies.
Common Applications:
  • Rain Water Trapping
  • Candy Distribution
  • Product Except Self
